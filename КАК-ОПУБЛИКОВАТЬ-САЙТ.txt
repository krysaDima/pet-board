КАК СДЕЛАТЬ САЙТ ДОСТУПНЫМ ВСЕМ В ИНТЕРНЕТЕ (надёжно)
====================================================

«С компьютера через туннель» часто не работает: провайдер, брандмауэр, нестабильный сервис.
Для настоящего доступа **всем пользователям** сайт выкладывают в интернет как **статический сайт** (папка после сборки).

Шаг 1 — собрать сайт на своём ПК
---------------------------------
Откройте терминал в папке pet-board и выполните:

  npm install
  npm run build

Появится папка **dist** — это готовый сайт (HTML, JS, CSS).

Шаг 2 — бесплатный хостинг (выберите один способ)
-------------------------------------------------

### A) Netlify (проще всего, без Git)

1. Зайдите на https://www.netlify.com и зарегистрируйтесь (можно через Google).
2. В кабинете: **Sites → Add new site → Deploy manually** (или «перетащите папку сюда»).
3. Перетащите **всю папку dist** в окно браузера.
4. Через минуту Netlify даст ссылку вида **https://случайное-имя.netlify.app** — её можно разослать всем.
5. В настройках сайта можно переименовать поддомен (Site settings → Domain management).

Файлы **public/_redirects** и **netlify.toml** уже в проекте — маршруты React (например /perederzhka) будут открываться после обновления страницы.

### B) Vercel (если привыкли к GitHub)

1. Залейте проект в репозиторий GitHub.
2. https://vercel.com → Import project → выберите репозиторий.
3. Сборка: **Build command** = `npm run build`, **Output** = `dist`.
4. Получите ссылку **https://ваш-проект.vercel.app**.

Файл **vercel.json** в проекте настроен под SPA.

### C) Cloudflare Pages

Аналогично: подключить Git или загрузить папку dist; корневая папка публикации — **dist**.

Шаг 3 — после каждого изменения кода
------------------------------------
Снова выполните **npm run build** и заново задеплойте папку **dist** (или пусть хостинг пересоберет из Git).

Почему не «просто открыть компьютер в интернет»
-----------------------------------------------
Домашний интернет обычно не даёт прямой вход на ваш ПК. Туннели (localtunnel, ngrok) — для кратких тестов и часто падают.
**Хостинг** — нормальный способ, как у настоящих сайтов.

Если всё же нужен туннель с ПК
-----------------------------
См. файл **доступ-из-интернета.txt**. Нужны ДВА окна: сначала **npm run dev**, потом **npm run tunnel**; ссылка — только из окна туннеля (https://….loca.lt), не localhost.
