ДОСТУП К САЙТУ ИЗ ИНТЕРНЕТА (ваш ПК как «сервер» для демо)
============================================================

Чтобы сайт был открыт ВСЕМ и стабильно — смотрите файл **КАК-ОПУБЛИКОВАТЬ-САЙТ.txt** (хостинг Netlify/Vercel после npm run build).
Ниже — временный вариант с ПК через туннель (часто ломается).

Ваш домашний интернет обычно НЕ даёт «белый» IP на компьютер напрямую.
Поэтому «чтобы зашли из интернета» делают туннель: внешний адрес ведёт на ваш localhost.

Вариант A — два окна (рекомендуется)
------------------------------------
1) Окно 1 — в папке pet-board выполните:
     npm run dev
   Дождитесь, пока сайт поднимется на порту 5173.

2) Окно 2 — в той же папке:
     npm run tunnel
   В консоли появится строка вида:
     your url is: https://xxxx.loca.lt
   Эту ссылку можно отправить другу — он откроет ваш dev-сайт из интернета.

3) Оба окна держите открытыми, пока нужен доступ.

Вариант B — один двойной щелчок
--------------------------------
  Запустите файл:  туннель-интернет.bat
  Откроются два окна: сначала dev, затем туннель. Ссылку берите из окна «tunnel».

Важно
------
• Это только для РАЗРАБОТКИ. Не храните пароли и секреты в таком режиме.
• Ссылка localtunnel меняется при каждом новом запуске (если не настраивать свой поддомен).
• Антивирус / брандмауэр могут спросить разрешение для Node.js — для туннеля нужен исходящий интернет.
• Если страница по ссылке не грузится — подождите 10–15 с после старта Vite и перезапустите npm run tunnel.

Альтернатива: ngrok (https://ngrok.com) — после регистрации: ngrok http 5173
(удобно, если localtunnel нестабилен у вашего провайдера).
